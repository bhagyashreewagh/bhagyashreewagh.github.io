The problem

Medical reports are written for specialists, not patients. A cardiology report, a neurology note, and a pulmonology finding can all be generated from the same hospital visit, but they're written in different vocabularies, for different audiences, and rarely synthesized into one coherent picture.

Upload a single PDF medical report. Six specialist AI agents analyze it in parallel and return a unified multidisciplinary diagnosis, each from their own domain perspective.

The six agents
CardiologyHeart function, blood pressure, cardiovascular risk factors
NeurologyNervous system findings, cognitive markers, neurological symptoms
PulmonologyLung capacity, respiratory function, airway findings
PsychologyMental health indicators, behavioral patterns, stress markers
EndocrinologyHormonal levels, metabolic markers, thyroid and glucose data
ImmunologyImmune response, allergy indicators, inflammatory markers
Architecture

The PDF is extracted using PyMuPDF, which handles scanned and digital reports. The raw text is passed to all six agents simultaneously using Python's ThreadPoolExecutor, so no agent waits for another. Each agent has a domain-specific system prompt that focuses its analysis on the findings relevant to its specialty.

Once all six agents complete their analysis, a final Multidisciplinary Agent synthesizes the individual outputs into a coherent diagnosis summary, identifying patterns across specialties that a single-domain analysis would miss.

Why parallel agents

A single generalist model reading the same report tends to prioritize certain findings over others depending on how the report is written. Running six focused agents in parallel forces the system to surface findings that might otherwise be overshadowed. The synthesis step then looks for cross-domain patterns, for example, where a metabolic finding from endocrinology reinforces a cardiovascular risk flag from cardiology, that only become visible when all perspectives are on the table at once.
