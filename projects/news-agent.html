The problem

Keeping up with a topic like AI or data science means reading a lot, and most of it is noise. Articles repeat each other, headlines are misleading, and there's no quick way to tell whether something is worth reading before you open it.

Type a topic. The agent fetches live headlines, summarizes each one with Claude, runs sentiment analysis, and returns a curated feed with scores, so you know what's worth reading before you click.

Pipeline
1
Fetch: Live headlines are pulled from Google News RSS for the user's topic. No API key required.
2
Summarize: Each article description is passed to Claude Haiku with a focused prompt to return a two-sentence summary. Fast and cheap per call.
3
Sentiment: VADER scores each article as Positive, Negative, or Neutral with a compound score. VADER is rule-based and runs locally with no latency.
4
Display: Results render as a clean feed with source, summary, sentiment badge, and a direct link to the full article.
Why VADER alongside Claude

Claude is used for summarization because it handles nuanced, domain-specific language well and produces consistently readable output. VADER handles sentiment because it's a rule-based lexicon model that runs instantly and locally, with no additional API call, no latency, no cost. The two models do genuinely different jobs: Claude understands the content, VADER classifies the tone. Using Claude for sentiment would be slower and more expensive with no meaningful accuracy gain for this use case.

Design choice: RSS over a news API

Early versions used the NewsAPI, which requires registration and has rate limits on the free tier. Switching to Google News RSS removed the dependency entirely, with no key, no rate limit, always fresh. The trade-off is less structured metadata per article, but for this use case (headline, description, source, link) RSS provides everything needed.
